git-notified — dropdown layout mockup

Four variants applying the research synthesis. Each frame is 380 px (real menubar popover width). Mock data: 3 repos, mix of review-requests, CI, comments, reviews.
Click a button to switch — or scroll to compare side-by-side. Try expanding PRs in V4.
V1 — Linear lean

ALL-CAPS muted repo headers, PR title primary, indented events with small type-icons. Pure spacing hierarchy, no borders. Closest to your Variant A request — less dense via typography, not less info.

V2 — Things calm

Maximum whitespace. PR title is the hero, max 2 events shown with "+N more" affordance. Most breathable; trades some scan-completeness for calm.

V3 — GitHub Notifications style

Every event is a row under its repo header — no PR-level grouping in the visual sense, just sorting. Hairlines between rows. Familiar from github.com/notifications.

V4 — Slack roll-up

PR card shows a one-line summary (chips). Click to expand events inline. Default-collapsed. Most compact baseline, deepest detail on demand.
